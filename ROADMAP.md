# 🗺️ Dycast Desktop 功能路线图

> 当前版本：**v1.3.0** | 更新日期：2026-06-10

---

## 项目现状

Dycast Desktop 是一个成熟的抖音直播弹幕桌面客户端（Tauri 2 + Vue 3），核心链路稳定：

| 能力 | 状态 |
|------|------|
| 弹幕实时接收（WebSocket 长连接） | ✅ |
| 消息分类展示（聊天/礼物/点赞/进场/关注） | ✅ |
| 虚拟滚动 + 3000 条上限 | ✅ |
| 富文本渲染（emoji、@提及、礼物图标） | ✅ |
| 消息转发（WebSocket relay 到外部服务） | ✅ |
| JSONL 本地录制 | ✅ |
| Session Cookie 登录（获取礼物数据） | ✅ |
| 断线自动重连（最多 3 次） | ✅ |
| 类型过滤（按消息类型显示/隐藏） | ✅ |
| 直播信息卡片（封面、标题、主播、统计） | ✅ |
| Tauri + 浏览器双平台抽象 | ✅ |
| CI/CD 自动发布（Win/Mac/Linux） | ✅ |

## 路线图

### 🔴 P0 — 体验补完

这些是基础体验的缺失，直接影响用户留存。

#### 1. 亮色/暗色主题切换

- **现状**：左右两列各自硬编码了亮色和暗色主题，用户无法全局切换
- **方案**：
  - 在 SCSS 中定义 CSS 变量（`--bg`, `--text`, `--border` 等），抽离两套主题
  - 在 `IndexView` 顶部或设置面板加一个 toggle 按钮
  - 偏好存入 `localStorage`，启动时读取
- **工作量**：小（1-2 天）

#### 2. 持久化设置记忆 ✅（已实现基础版）

- **现状**：每次启动需重新输入房间号、中继地址、session cookie
- **方案**：
  - 使用 `localStorage`（浏览器模式）或 Tauri store plugin（桌面模式）
  - 记忆项：上次房间号、中继 WS 地址、窗口尺寸/位置、主题偏好、消息类型过滤状态
  - 启动时自动回填，但不自动连接（安全考虑）
- **工作量**：小（1 天）
- **实现**：v1.3.0 已实现设置面板（齿轮按钮），支持记住房间号、转发地址、自动更新开关

#### 3. 桌面通知

- **现状**：窗口最小化后无法感知重要事件
- **方案**：
  - 使用 Tauri notification plugin 或 Web Notification API
  - 触发场景：大礼物（如嘉年华、火箭）、新关注、直播结束
  - 可配置：通知开关、触发阈值（礼物价值门槛）
- **工作量**：中（2-3 天）

---

### 🟡 P1 — 信息增强

这些功能放大核心价值——不只是"看到弹幕"，而是"看懂弹幕"。

#### 4. 实时消息统计面板

- **现状**：只有原始消息流，无聚合数据
- **方案**：
  - 新增 `StatsPanel.vue` 组件，放在左侧栏或弹出面板
  - 统计维度：
    - 本场总弹幕数、独立发言用户数
    - 礼物收入排行（Top 10 送礼用户，按价值降序）
    - 弹幕频率趋势（简易折线图，每分钟弹幕数）
    - 最活跃观众（发言次数排行）
  - 数据在 `IndexView.handleMessages()` 中增量更新，无需额外请求
- **工作量**：中（3-5 天）

#### 5. 礼物排行榜 UI

- **现状**：`WebcastRoomUserSeqMessage` 和 `WebcastRoomRankMessage` 的解码逻辑已就绪，但只作为普通消息展示，无可视化
- **方案**：
  - 新增 `GiftRank.vue` 组件，放在左侧栏
  - 展示：排名、用户头像、昵称、贡献值、礼物图标列表
  - 实时更新排名变化（上升/下降箭头动画）
- **工作量**：中（2-3 天）

#### 6. 弹幕搜索与过滤

- **现状**：3000 条历史只能滚动浏览，无法快速定位
- **方案**：
  - 在 CastList 头部加搜索框
  - 支持：按用户昵称搜索、按关键词搜索（文本内容）、按消息类型过滤
  - 搜索结果高亮，上下导航跳转
  - 可配合时间范围过滤
- **工作量**：中（2-3 天）

---

### 🟢 P2 — 场景扩展

这些功能打开新的使用场景和用户群。

#### 7. 多房间同时连接

- **现状**：单例 `DyCast`，一次只能连接一个直播间
- **方案**：
  - 重构为 `CastManager` 管理多实例，每个实例独立 WebSocket + 消息队列
  - UI 改为 Tab 切换，每个 Tab 一个房间
  - 录制/转发独立开关（某些房间只看不录）
  - 内存管理：每个房间独立 3000 条上限，总计不超过 10000
- **工作量**：大（1-2 周）

#### 8. 音效提示

- **现状**：纯视觉，无声音反馈
- **方案**：
  - 使用 Web Audio API 播放短音效
  - 内置几种默认音效（礼物、关注、消息）
  - 支持用户自定义音效文件
  - 可配置：音量、触发事件类型
- **工作量**：小（1-2 天）

#### 9. OBS 叠加层模式（差异化功能）

- **现状**：只能作为独立窗口使用
- **方案**：
  - 新增"直播模式"：透明无边框窗口 + 始终置顶
  - 弹幕以半透明浮层形式叠加在直播画面上方
  - 可调：透明度、字体大小、滚动速度、显示区域
  - 主播可通过 OBS 窗口捕获直接使用
  - 这是区别于同类弹幕工具的关键差异化功能
- **工作量**：大（1-2 周）

---

### 🔵 P3 — 工程完善

这些保持项目长期健康，不直接产生用户价值但降低维护成本。

#### 10. 自动更新检查 ✅（已实现）

- **方案**：接入 Tauri updater plugin，发布新版本时自动弹窗提示
- **工作量**：小（1 天）
- **实现**：v1.3.0 已集成 `tauri-plugin-updater`，检查 GitHub Releases，弹窗提示下载安装

#### 11. 系统托盘

- **方案**：最小化到托盘，后台持续录制/转发，右键菜单快速操作
- **工作量**：中（2-3 天）

#### 12. 错误恢复改进

- **现状**：写死 3 次重连，无退避策略
- **方案**：指数退避（1s → 2s → 4s → 8s → 16s，上限 5 分钟），增加重连成功率
- **工作量**：小（半天）

## 不会做的

以下功能明确在当前范围内不做（欢迎 PR）：

- ❌ **发送弹幕/礼物** — 涉及抖音 API 风控，法律风险高，且与项目定位（弹幕接收工具）不符
- ❌ **弹幕自动回复机器人** — 同上有风控和法律风险
- ❌ **付费功能/商业化** — 项目保持 MIT 开源
- ❌ **移动端（iOS/Android）** — Tauri 2 目前仅支持桌面端
