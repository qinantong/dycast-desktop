# Dycast Desktop 功能路线图

> 当前版本：**v1.3.3** | 更新日期：2026-06-11

---

## 项目现状

Dycast Desktop 是一个面向**弹幕游戏开发者**的抖音直播弹幕桌面客户端（Tauri 2 + Vue 3），核心链路稳定：

| 能力 | 状态 |
|------|------|
| 弹幕实时接收（WebSocket 长连接） | ✅ |
| 消息分类展示（聊天/礼物/点赞/进场/关注） | ✅ |
| 虚拟滚动 + 3000 条上限 | ✅ |
| 富文本渲染（emoji、@提及、礼物图标） | ✅ |
| 消息转发（WebSocket relay 到外部服务） | ✅ |
| 转发消息类型自定义过滤 | ✅ |
| JSONL 本地录制 | ✅ |
| Session Cookie 登录（获取礼物数据） | ✅ |
| 断线自动重连（最多 3 次） | ✅ |
| 亮色/暗色主题切换 | ✅ |
| 持久化设置记忆（房间号、转发地址、主题） | ✅ |
| 自动更新检查（Tauri updater） | ✅ |
| 类型过滤（按消息类型显示/隐藏） | ✅ |
| 直播信息卡片（封面、标题、主播、统计） | ✅ |
| Tauri + 浏览器双平台抽象 | ✅ |
| CI/CD 自动发布（Win/Mac/Linux） | ✅ |
| 系统字体替代自定义字体（安装包 -2.5MB） | ✅ |

## Backlog

### 🔥 高优先级

- [ ] **录制回放** — 将 `.jsonl` 录制文件按原始时间戳重放至 WebSocket relay，游戏开发者无需真开播即可反复测试游戏逻辑
- [ ] **连接重连指数退避** — 当前写死 3 次立即重连，改为 1s→2s→4s→8s→16s 退避，上限 5 分钟
- [x] **字体体积优化** — 移除 DouyinSansBold（2MB）+ MaoKenWangXingYuan（3MB），改用系统默认字体，MSI 安装包减小约 2.5MB

### 🟡 中优先级

- [ ] **桌面通知** — 窗口最小化后通过 Tauri notification plugin 或 Web Notification API 推送大礼物/新关注/下播通知，支持通知开关和礼物价值阈值配置
- [ ] **HTTP 本地查询接口** — 在 localhost 提供 REST 接口（如 `GET /messages?count=50`），游戏后端可直接拉取消息，无需走 WebSocket
- [ ] **房间收藏/预设** — 保存多个房间号 + 别名，下拉快速切换，方便同时维护多个直播间的开发者
- [ ] **实时消息统计面板** — 聚合数据面板：本场弹幕总数、独立用户数、礼物收入 Top 10、弹幕频率趋势、最活跃观众
- [ ] **礼物排行榜 UI** — 后端解码已就绪，需新增 `GiftRank.vue` 组件展示实时排名变化

### 🟢 低优先级

- [ ] **弹幕搜索与过滤** — 在 CastList 中按用户昵称/关键词搜索历史消息，结果高亮，上下导航跳转
- [ ] **音效提示** — Web Audio API 播放短音效，内置默认音效 + 用户自定义，可配置音量和触发事件
- [ ] **系统托盘** — 最小化到托盘，后台持续录制/转发，右键菜单快速操作
- [ ] **多房间同时连接** — 重构为 `CastManager` 管理多实例，UI Tab 切换，每个房间独立上限 3000 条
- [ ] **Release 自动化改进** — `releaseBody` 从 workflow 抽离为独立 `RELEASE_NOTES.md`，或从 commit 自动生成

## 技术债务

- [ ] **补充单元测试**（当前覆盖率 0%）— 至少为 core/protobuf 解码加测试，防抖音协议变更
- [ ] **`model/base.ts` 仍有大量内容** — 约 15K 行，继续拆分到 `messages/` 目录
- [ ] **`any` 类型收窄** — 当前 24 处 `any` 可部分替换为具体接口
- [ ] **依赖更新** — `vue-virtual-scroller` 2.0.0-beta.8 → 3.x，Vite 6.4.2 有小版本更新
- [ ] **录制浏览器兼容** — 浏览器模式依赖 File System Access API（Chrome 独占），Safari/Firefox 需回退

---

## 已移除

- ❌ **OBS 叠加层模式** — 抖音直播伴侣已自带弹幕叠加层，此功能不再具备差异化价值

## 不会做的

- ❌ 发送弹幕/礼物 — 涉及抖音 API 风控，法律风险高
- ❌ 弹幕自动回复机器人 — 同上
- ❌ 付费功能/商业化 — 项目保持 MIT 开源
- ❌ 移动端（iOS/Android）— Tauri 2 仅支持桌面端
